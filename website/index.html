Why Popush

The deploy button your server always needed.

No web dashboard, no agent to install on the box, no yaml to babysit. Just SSH, your git repo, and one honest button.

Honest status

Green when your site actually answers over HTTPS, amber when Popush isn't sure. It shows the truth a visitor sees — never an optimistic guess.

One button, whole deploy

Commit → push → pull → build → restart → verify. Every step streams its logs live, waits for the site to come healthy, and cancels the moment you say so.

Your stack, your rules

Docker Compose, systemd, pm2, or a plain static site. Popush drives them over ordinary SSH — there's nothing extra to install on the server.

Local & private

Your servers, keys, and config live in a file on your machine. Nothing is uploaded, tracked, or phoned home — ever. Verify it; the source is open.

Three steps

From fresh install to shipped.

01

Add your server

Point Popush at your VPS with its SSH key. It verifies the host key on first connect — a changed key is refused, so no silent man-in-the-middle.

02

Add a site

Give it the folder on the server and pick the type — Docker, systemd, pm2, or static. Add a URL and Popush watches its real status.

03

Ship it

Press the button. Watch the pipeline stream, wait for the health check to go green, and land on "Shipped and live." That's the whole workflow.
